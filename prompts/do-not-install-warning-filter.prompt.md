# Lovable Do-Not-Install Warning Filter

Trigger: Use when saving project instructions, adapter notes, repo files, or prompt-library entries.

Do not save the following as persistent doctrine:

1. Lovable can produce a native iOS/App Store binary by itself.
2. Security View is a final security certification.
3. Lovable can import and continue the existing GitHub repo.
4. Lovable Cloud is the safest default for Kesher.
5. PWA/installability works without smoke tests.
6. Capacitor/iOS wrapper is production-ready without device tests.
7. Paid-plan controls, private beta controls, or workspace privacy settings are available without live workspace verification.
8. Client-side checks are enough for messaging, moderation, reports, billing, or admin routes.
9. Real user messages, photos, abuse reports, or production PII may be pasted into prompts.
10. Lovable-specific mode names belong in global project doctrine.
