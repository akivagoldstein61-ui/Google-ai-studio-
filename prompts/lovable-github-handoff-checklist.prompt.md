# Lovable GitHub Handoff Checklist

1. Do not assume existing repo import.
2. Use disposable sync repo first.
3. Confirm default-branch sync behavior.
4. Confirm non-default branch behavior.
5. Document sync repo, branch, commit SHA, and limitations.
6. Do not rename/move/delete linked repo after connection.
7. Promote code into canonical repo through PR only.
8. Require CODEOWNERS review, required checks, tests, secret scan, safety/privacy review, rollback notes.
9. Do not set secrets or push production config through generated code.
