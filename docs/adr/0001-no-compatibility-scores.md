# ADR 0001: No Compatibility Scores

## Status

Accepted

## Context

Kesher can help users notice shared values and communication patterns, but dating compatibility cannot be responsibly reduced to a score or percentage. Scores create false precision, hidden ranking pressure, and over-trust in model output.

## Decision

Kesher personality and match features must not produce compatibility scores, match percentages, soulmate claims, destiny claims, desirability tiers, or hidden ranking explanations.

"Why This Match" may provide short, transparent reasons based on visible profile fields. Compatibility reflection may provide shared strengths, possible friction, a question to explore, a micro-habit, and a gentle boundary after mutual opt-in.

## Consequences

- Product copy must use uncertain, reflective language.
- AI validators must block score-like and destiny-like output.
- Ranking logic, if it exists elsewhere, must not be exposed as personality truth.
- Tests should cover score, percentage, soulmate, perfect-match, and doomed/incompatible language.
